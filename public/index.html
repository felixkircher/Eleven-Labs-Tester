📖 How-To & Dokumentation

🚀 Setup & Installation

Hinweis: Dieser Schritt ist nur bei lokaler Nutzung ohne die App wichtig. Ansonsten weiter mit dem nächsten Schritt.
Wichtig für Devs: Wenn andere Anwendungen den Port 3000 bereits verwenden, wird die App nicht korrekt starten. Bitte Port 3000 freihalten und App erneut starten.

Der Agent Tester läuft als eigenständiger Node.js-Server — kein Node-RED, kein externes Backend nötig.

Voraussetzungen

  • Node.js ≥ 18
  • ElevenLabs-Account mit zwei konfigurierten Conversational AI Agents
  • API Key (xi-…) aus dem ElevenLabs-Dashboard

Starten

git clone https://github.com/felixkircher/Eleven-Labs-Tester.git cd eleven-labs-tester npm install npm start

Der Server läuft dann auf http://localhost:3000. Der Port kann über die Umgebungsvariable PORT geändert werden.

Optionale Vorkonfiguration

Erstelle eine config.local.json im Projektordner, damit API Key und Agent-IDs beim Neustart automatisch geladen werden:

{ "apiKey": "xi-…", "docAgentId": "agent_xy", "patientAgentId": "agent_yz", "maxTurns": 30, "silenceTime": 4000 }
⚠ Nicht committen! Diese Datei enthält sensible Daten. In .gitignore eintragen: config.local.json und /data
Hinweis: Beispiel-Dateien liegen in git jeweils mit example.<Dateiname> kopiert, wenn sie im Browser eingegeben werden.

Die Felder in der linken Sidebar steuern, wie der Testlauf abläuft.

📋 Base-Prompt (Patient)

Der Base-Prompt enthält allgemeine Verhaltensregeln, die für alle Szenarien gelten — z. B. natürlich sprechen, Infos erst auf Nachfrage nennen, Gespräch korrekt beenden.

Funktionsweise

  • Der Toggle oben rechts aktiviert/deaktiviert den Base-Prompt global für alle Szenarien
  • Pro Szenario kann er mit use_base_prompt: false individuell abgeschaltet werden
  • Der finale Prompt an den Agenten lautet: Base-Prompt + --- + Szenario: [Prompt]

Wann deaktivieren?

Für Off-Topic- oder Robustheitstests (z. B. „Frag nach Gehalt", „Datenschutz-Angriff") macht es oft Sinn, den Base-Prompt abzuschalten — der Agent soll dann ungelenkt agieren.

Tipp: Szenarien mit deaktiviertem Base-Prompt erscheinen leicht transparent in der Liste als visueller Hinweis.

🧪 Szenarien verwalten

Neues Szenario anlegen

Klicke auf + oben rechts in der Szenarienliste. Felder:

  • Name — Anzeigename (z. B. 01 · Neupatient – Terminvereinbarung)
  • Patienten-Prompt — Welche Rolle soll der Patient-Bot übernehmen? Persönlichkeit, Anliegen, Verhalten.
  • Base-Prompt verwenden — Toggle, ob der globale Base-Prompt für dieses Szenario aktiv ist.

Szenario bearbeiten

Klicke auf den Card-Text (nicht auf einen der Buttons) — das Bearbeitungs-Modal öffnet sich.

Import / Export

↑ importiert eine .json-Datei und fügt die Szenarien zur bestehenden Liste hinzu. ↓ exportiert alle aktiven (nicht archivierten) Szenarien.

JSON-Format:

[ { "name": "Terminvereinbarung", "prompt": "Du bist Max Mustermann…" }, { "name": "Off-Topic Test", "prompt": "Du fragst nach anderen Dingen…", "use_base_prompt": false } ]

☑ Szenarien auswählen

Nicht immer sollen alle Szenarien in einem Testlauf ausgeführt werden. Die Checkbox-Auswahl ermöglicht das gezielte Testen einzelner Szenarien.

Bedienung

  • Jeder aktive Szenario-Card hat eine Checkbox links — nur angehakte Szenarien laufen bei „▶ Alle testen"
  • ☑ oben → alle auswählen
  • ☐ oben → alle abwählen

Feedback

  • Das Badge X / Y ausgewählt erscheint, sobald die Auswahl unvollständig ist
  • Abgewählte Cards erscheinen transparent
  • Der Start-Button zeigt dynamisch: ▶ Auswahl testen (X) oder ▶ Alle testen (X)
Hinweis: Der ▶-Button im Card startet ein einzelnes Szenario sofort — unabhängig von der Checkbox-Auswahl.

⊟ Szenarien archivieren

Szenarien, die aktuell nicht gebraucht werden, aber nicht gelöscht werden sollen, können archiviert werden.

Archivieren

Klicke auf ⊟ im Card → das Szenario verschwindet aus der aktiven Liste.

Archiv anzeigen

Am Ende der Szenarienliste erscheint ▶ Archiv (N) — klicke darauf, um die archivierten Szenarien einzuklappen/aufzuklappen.

Wiederherstellen & Löschen

  • ↩ stellt ein Szenario zurück in die aktive Liste
  • × löscht es endgültig

Was archivierte Szenarien nicht tun

  • Sie nehmen nicht an Testläufen teil (auch wenn „Alle testen" gewählt wird)
  • Sie werden beim JSON-Export nicht mitexportiert

▶ Test ausführen

⚠ Wichtig: Der Patienten-Agent-Prompt wird bei jedem Szenario dauerhaft in ElevenLabs überschrieben. Nach dem Test hat der Agent den Prompt des zuletzt ausgeführten Szenarios.
⚠ Wichtig: Es kann immer nur ein Szenario gleichzeitig in ElevenLabs getestet werden, da sonst die Gespräche mischen würden.

Alle / Auswahl testen

Klicke ▶ Alle testen oder ▶ Auswahl testen (X). Der Server führt folgende Schritte pro Szenario aus:

  • Patienten-Agent-Prompt via ElevenLabs PATCH-API aktualisieren
  • Beide Agents per WebSocket verbinden
  • Praxis begrüßt zuerst — Gespräch beginnt
  • Nachrichten werden hin- und hergeleitet bis ein Abbruchkriterium greift
  • Transcript wird gespeichert, 4 Sekunden Cooldown, dann nächstes Szenario (Hier empfele ich 4 Sekunden, wenn der Agent mit TTS läuft und auch "sprechen" kann)

Einzelnes Szenario

Klicke auf ▶ im jeweiligen Card — läuft sofort, Checkbox wird ignoriert.

Szenario überspringen

Während ein Test läuft, erscheint ⭭ — klicke ihn, um das aktuelle Szenario abzubrechen und zum nächsten zu springen. Das abgebrochene Szenario wird als skipped gespeichert.

Test stoppen

■ Stopp beendet den gesamten Testlauf sofort. Das laufende Szenario wird abgebrochen, der bisherige Transcript gespeichert.

⛔ Abbruchkriterien

Jedes Szenario endet automatisch, sobald eines dieser Kriterien erfüllt ist:

GrundBedeutungWertung
patient_disconnectedPatient hat aufgelegt (WebSocket geschlossen)✓ Normal
praxis_disconnectedPraxis hat aufgelegt (WebSocket geschlossen)✓ Normal
max_turnsMaximale Turnanzahl erreichtNeutral
loop_detectedLoop erkannt — Agent wiederholt sich zu oft⚠ Warn
timeoutSzenario läuft > 5 Minuten ohne Ende⚠ Warn
skippedManuell übersprungen via ⭭⚠ Warn
ws_errorWebSocket-Verbindungsfehler✗ Fehler

Loop-Erkennung

Der Server erkennt drei Muster automatisch:

  • 3× identisch — drei aufeinanderfolgende Nachrichten (beliebiger Sprecher) sind gleich
  • Per-Agent-Wiederholung — ein Agent wiederholt seine letzten 3 eigenen Nachrichten
  • Ping-Pong — das Muster A–B wiederholt sich 3× in den letzten 6 Nachrichten
Tipp: Loops deuten oft auf zu kurze Silence-Zeit oder auf einen schlecht konfigurierten Agenten hin.

📊 Transcript, Log & Historie

Transcript-Tab

Live-Ansicht des Gesprächs. Praxis-Nachrichten erscheinen links, Patienten-Nachrichten rechts. Systemnachrichten (Szenario-Ende, Ergebnis) erscheinen zentriert.

System Log-Tab

Technisches Log: Verbindungsaufbau, Prompt-Updates, Loop-Erkennungen, Fehler — mit Zeitstempel.

Historie-Tab

Alle abgeschlossenen Test-Runs werden automatisch unter /data/ gespeichert.

  • Klicke auf einen Run → vollständiger Transcript wird angezeigt
  • ⬇ exportiert den Run als .txt-Datei
  • × löscht den Run
  • ← Zurück kehrt zur Historie-Liste zurück

Export

⬇ Export oben rechts in der Toolbar exportiert den aktuell sichtbaren Transcript (oder den gerade betrachteten historischen Run).

💡 Tipps & Hinweise

  • Silence-Zeit erhöhen (z. B. 4000 ms), wenn Agents abgeschnitten werden oder Antworten unvollständig wirken
  • Max Turns senken (z. B. 10–15) für schnellere Testläufe bei einfachen Szenarien
  • Szenarien deselektieren, um nur bestimmte Fälle zu testen, ohne alle laufen zu lassen
  • Archiv nutzen, um Szenarien temporär aus dem Testlauf auszublenden, ohne sie zu verlieren
  • Off-Topic-Szenarien (🛡) mit use_base_prompt: false markieren, damit der Patienten-Bot ungelenkt reagiert
  • Loop-Erkennungen als Warnsignal: Sie deuten oft auf Konfigurationsprobleme beim Agent hin
  • Zwischen Szenarien wartet der Server automatisch 2 Sekunden Cooldown, damit die Agent-Konfiguration übernommen wird
Datenschutz: Der API Key wird nur im sessionStorage des Browsers gespeichert und verlässt das Gerät nur in Richtung ElevenLabs-API. Die config.local.json wird serverseitig gelesen — der API Key erscheint nicht im HTML-Quellcode.
⚠ Agent-Prompt: Der Patienten-Agent-Prompt wird dauerhaft in ElevenLabs überschrieben, nicht nur temporär. Nach einem Testlauf hat der Agent den Prompt des zuletzt getesteten Szenarios.
